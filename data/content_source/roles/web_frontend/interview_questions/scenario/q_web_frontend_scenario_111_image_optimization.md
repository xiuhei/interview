# 在真实场景里你会怎样使用 图片优化？

## question
如果在项目里遇到和 图片优化 相关的问题，你会如何判断是否采用它，并如何验证效果？

## reference_answer
图片优化 是 Web 前端 面试里常见的高频知识点，回答时应覆盖定义、机制、适用场景、边界条件和代价。 场景题的关键是说明为什么这样选、怎么验证结果以及不选时的替代方案。

## key_points
- 先讲清 图片优化 的定义或问题背景
- 再说明机制、限制和真实场景
- 补充项目证据、指标或验证方式
- 明确边界条件与风险点

## common_mistakes
- 只背结论，不解释为什么
- 不说明适用边界和代价
- 无法给出真实项目或验证证据

## scoring_rubric
- 90+：原理、取舍、指标和项目复盘都能讲清
- 75+：原理与场景清楚，但细节略少
- 60+：只会背概念，没有真实落地证据
- 40-：概念混乱或无法自洽

## follow_up_questions
- 图片优化 在你的项目里最难的取舍是什么？
- 如果结果不符合预期，你会怎样验证是不是 图片优化 引起的？
- 图片优化 和 设计系统 的边界应该怎么划分？

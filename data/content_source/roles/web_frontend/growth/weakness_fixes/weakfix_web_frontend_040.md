# 薄弱项补强：故障复盘不完整

## diagnosis
你在 Web 前端 面试或练习里经常表现出“故障复盘不完整”这个问题。

## improvement_advice
回答时固定加上结论、原因、做法、结果四段；如果是项目题，再补职责边界和验证方式。

## recommended_practice
连续完成 3 道对应专项题，每题都做 1 次录音复盘，并记录改进前后的差异。

## expected_outcome
回答会更完整、更有证据，也更容易体现岗位匹配度。

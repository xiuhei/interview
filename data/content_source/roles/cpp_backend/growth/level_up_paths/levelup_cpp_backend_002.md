# 进阶路径：从 API 使用到机制解释

## diagnosis
从 API 使用到机制解释 是很多 C++ 后端 候选人从初级走向中级时的关键跃迁。

## improvement_advice
每个阶段都要补机制、补取舍、补项目证据，不能只扩技术栈广度。

## recommended_practice
为该阶段设计 2 周专项计划，完成题目训练、知识整理和项目复盘。

## expected_outcome
回答会从执行层升级为工程判断层。
